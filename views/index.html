Okta Password Import Inline Hook: Verify User Credentials and Import Password to Okta Org

Complete overview and step-by-step guide available at Password Import Inline Hook

A Node.js application written to receive Password Import Inline Hook requests from Okta, parse password credentials, and, if verified, import passwords back into the Okta org.

At a high-level, the following workflow occurs:

  • User profiles are imported into an Okta org that use the Password Import Hook.
  • The Password Import Inline Hook triggers on first sign-in request by a user.
  • The external service evaluates the Password Import Inline Hook request's user credentials against the data store.
  • If the credentials are verified, the external service responds to Okta with a command to import the password and sign in the user.
  • If the credentials are not verified, the user is not signed-in or imports the password.

See Password Import Inline Hook Reference for reference JSON documentation.
